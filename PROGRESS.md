## Design Decisions Made

| Decision | Rationale |
|----------|-----------|
| Flat `notebooks.json` with nested sections/pages | Simplest possible storage â€” no DB, easy to inspect/edit manually |
| `localStorage` for backend URL | Persists across sessions; iPad and laptop each store their own preference |
| `key={page.id}` on the tldraw component (planned) | Forces full remount on page switch â€” clean snapshot load/save lifecycle without managing imperative editor state |
| PDF rendered client-side via PDF.js | No server-side PDF rendering dep; keeps backend pure Python/Pillow |
| `vite-plugin-pwa` for service worker | Generates SW automatically from build output; much more reliable than a hand-written SW for Vite apps |
| `pageScreenshots` as a `MutableRefObject<Map>` | Avoids triggering re-renders on every screenshot write; the sync hook reads it imperatively |
| PNG icons generated by Pillow (not SVGâ†’PNG tool) | No `cairosvg` or browser-based conversion needed; pure Python draw calls |
| CSS variables on `[data-theme]` attribute | Works with SSR if ever needed; trivial to toggle from JS |
