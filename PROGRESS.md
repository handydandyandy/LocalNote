## Design Decisions Made

| Decision | Rationale |
|----------|-----------|
| Flat `notebooks.json` with nested sections/pages | Simplest possible storage — no DB, easy to inspect/edit manually |
| `localStorage` for backend URL | Persists across sessions; iPad and laptop each store their own preference |
| `key={page.id}` on the Canvas component | Forces full remount on page switch — clean snapshot load/save lifecycle without managing imperative editor state |
| PDF rendered client-side via PDF.js | No server-side PDF rendering dep; keeps backend pure Python/Pillow |
| PDF background shapes stripped from snapshot before save | Keeps canvas JSON files small; PDF re-renders from the backend URL on each mount |
| `meta: { isPdfBackground: true }` on PDF shapes/assets | Allows filtering them from saved snapshot without affecting user-drawn content |
| `vite-plugin-pwa` for service worker | Generates SW automatically from build output; much more reliable than a hand-written SW for Vite apps |
| `pageScreenshots` as a `MutableRefObject<Map>` | Avoids triggering re-renders on every screenshot write; the sync hook reads it imperatively |
| PNG icons generated by Pillow (not SVG→PNG tool) | No `cairosvg` or browser-based conversion needed; pure Python draw calls |
| CSS variables on `[data-theme]` attribute | Works with SSR if ever needed; trivial to toggle from JS |
| `editor.user.updateUserPreferences({ colorScheme })` (not `isDarkMode`) | `isDarkMode` doesn't exist in tldraw 2.4 — correct property is `colorScheme: 'dark' \| 'light' \| 'system'` |
| `pdfjs-dist` excluded from Vite dep optimization | pdfjs uses dynamic `require()` internally which breaks Vite's pre-bundler; exclusion lets it load natively |
| PDF.js worker via `new URL('...', import.meta.url)` | Vite-aware pattern; correctly emits the worker file into dist and resolves its URL |

## iPad Install (one-time)

1. Connect iPad to same WiFi as laptop
2. Open Safari → `http://<laptop-ip>:8000`
3. Share sheet → "Add to Home Screen"
4. App opens full-screen with no browser chrome

For a stable URL without router admin access, install Tailscale on both devices (see deployment notes in CLAUDE.md).

## How to Run

**Dev** (two terminals):
```bash
cd backend && python main.py    # FastAPI on :8000
cd frontend && npm run dev      # Vite on :5173 — set backendUrl to http://localhost:8000 in Settings
```

**Production** (single port, iPad-ready):
```bash
python setup.py                 # generate icons (one-time)
cd frontend && npm install      # install deps (one-time)
bash scripts/build.sh           # builds frontend → starts backend on :8000
# iPad opens http://<laptop-ip>:8000 — no backendUrl config needed
```
